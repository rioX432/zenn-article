# Writing Guide

記事執筆時の詳細ガイドライン。CLAUDE.mdの補足として参照する。

## 記事構成

### セクション構造

セクション間に水平線（`---`）を入れない。見出しで自然に区切る。

```markdown
## セクション1

内容...

## セクション2

内容...
```

### 段落

3〜5行程度の短さを保つ。長い段落は分割する。

## 強調・引用の使い分け

- **太字** — 段落内のキーワード強調に使う
- 引用（`>`） — 外部ソースからの引用にのみ使う。内容を目立たせるための装飾には使わない

```markdown
<!-- 良い -->
Kotlinの哲学は**安定性と一貫性**を重視している。

<!-- 避ける -->
> Kotlinの哲学は安定性と一貫性を重視している。
```

## 文体

- 自然な口語体。難しい横文字や堅い表現は避ける
- 「まず」「次に」「最後に」の多用は機械的に見えるので避ける
- セクションタイトル全部に絵文字をつけるのはAIっぽいのでNG。文中に散らすのはOK
- 試行錯誤・ハマったポイント・失敗談を入れると人間味が出る
- 断言的すぎる表現を避け、「〜の環境では」「私の場合は」など主観を明示する

## 人物への言及

X（旧Twitter）のハンドルをリンクで入れる。

```markdown
**名前（[@handle](https://x.com/handle)）**
```

## 画像キャプション

画像の下にキャプションが必要な場合はHTMLで。

```markdown
![alt](/images/screenshot.png)
*キャプションテキスト*
```

## 公開前チェックリスト

- [ ] slugが12〜50文字（a-z0-9, ハイフン, アンダースコアのみ）
- [ ] topicsが最大5つ
- [ ] 画像が3MB以下・絶対パスで参照している
- [ ] セクション間に `---` を入れていない
- [ ] 引用（`>`）を装飾目的で使っていない
- [ ] `docs/BLOG_EVALUATION_PROMPT.md` で評価し、7項目すべて4.0以上
- [ ] `published: true` に変更した

## 品質評価

公開前に `docs/BLOG_EVALUATION_PROMPT.md` の評価プロンプトで記事を評価する。

**目標**: 7項目すべてで **4.0以上**

特に重視する項目:
- **人間らしさ** — 失敗談や試行錯誤が含まれているか
- **実践応用性** — 読者が実際に試せるコードや手順があるか
- **炎上リスク** — 個人ブログとはいえ、誤解される表現がないか
